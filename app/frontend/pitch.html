PulseTraffic

Smart Traffic Signal Optimization — Karlsruhe

The Problem

86% of Karlsruhe's monitored intersections have traffic too unpredictable for fixed signal timing.

Peak hour (17:00) sees 2x more traffic than off-peak — but signals never adapt.

The Data

304,018Sensor Readings
50IoT Sensors
6Days of Data
3,914Road Nodes (OSMnx)

The Solution: Green Wave

Synchronize traffic lights along 5 corridors (2-3km each) so drivers hit consecutive green lights without stopping.

A hero vehicle drives 6.6km from Spassbecken to Heidesee on the shortest road, demonstrating the green wave in action.

The Impact

17%Fewer Vehicles Stopped

Tech Stack

Python / FastAPI WebSocket 10fps OSMnx Road Graph SQLite (304K rows) Leaflet.js Dark Map Dijkstra Routing Docker Real IoT Sensor Data

How It Works

1. Real Data → Database

304,018 sensor readings from 50 traffic sensors imported into SQLite. Vehicle types: car, motorbike, truck, bicycle.

2. Road Network Simulation

750 constant vehicles drive on 3,914 real Karlsruhe intersections (OSMnx). Traffic lights cycle at 749 intersections. Vehicles stop at red, queue, resume at green.

3. Green Wave Optimization

Dijkstra finds shortest road paths between high-traffic sensors. 5 corridors identified. Signal phases synchronized with travel-time offsets at 30km/h. 30% of vehicles rerouted to corridor roads.

4. Live Dashboard

WebSocket streams vehicle positions at 10fps. Narrative UI: Problem → Hero Metric → One-Click Solution → Before/After Impact.
